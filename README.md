# CityWatch

> See what's happening around you. For real. With sources.

CityWatch is a real-time safety awareness app that helps users stay informed about incidents in their area. It features a live interactive map with sourced incident data, AI-powered chat for natural-language safety queries, automated area safety briefs, and community reporting with verification — all designed to make neighborhoods safer and more transparent.

Built for the **Claude Builder Club Hackathon at HackASU** (March 20-22, 2026).

## Features

### Live Incident Map

- Real-time interactive map powered by **Mapbox GL** showing incidents from multiple verified sources (police reports, news, community reports)
- Filter by category (theft, assault, vandalism, harassment, vehicle break-in, disturbance, infrastructure) and time range
- Incident clustering and detail views with source attribution

### AI-Powered Chat

A hybrid AI assistant powered by **Claude** with two intelligent modes:

**How it works:**

- **Fast lane** — For simple questions like _"What's happening near downtown?"_, it takes a fast path: geocodes the location, queries our real incident database, and Claude summarizes the results in one streaming call. Answers in **1-2 seconds**
- **ReAct reasoning lane** — For people questions like _"Where is Anirudh, is he safe?"_, Claude enters a multi-step reasoning loop where it decides what tools to call and in what order based on the question. Handles complex multi-person queries in **3-4 seconds**
- **Zero hallucination** — The LLM never generates fake data. Every single incident in every response comes from our real database (Phoenix PD police data via TinyFish). Claude only summarizes, explains, and reasons over real records
- **Hybrid cost savings** — This two-lane approach saves **5-10x on API costs** compared to running ReAct for everything, while keeping complex queries fully capable

**Technical highlights:**

- **Pure Python classifier** — A rule-based router (no LLM call) decides whether a question needs simple lookup or ReAct reasoning, adding zero latency to the routing decision
- **Context chaining** — The session tracks which person, which location, which time range, and which lane was used last, so follow-ups like _"Is he safe?"_ and _"What about her?"_ work naturally without repeating context
- **Server-side card building** — Incident cards are built from real database records, never generated by the LLM, preventing hallucinated data from ever reaching the UI
- **Pre-fetching** — Before the ReAct loop starts, we already fetch all group members and their live locations, so Claude doesn't waste reasoning steps on lookups
- **Real data pipeline** — We scraped Phoenix PD open data using **TinyFish** (hackathon sponsor), not fake seed data. Every incident is real, geocoded, and categorized

### Area Safety Briefs

- Automated safety summaries for any area, generated by Claude
- Cached for performance, refreshed on schedule

### Groups & Live Location Sharing

- Create family or friend groups with invite codes
- Real-time location sharing within groups via WebSocket
- See group members on the map with staleness indicators

### Community Reporting

- Submit incident reports with location, category, and details
- Verification pipeline for community-sourced data

### User Profiles & Saved Places

- Save home, work, school, and favorite locations
- Onboarding flow with age-band selection and notification preferences

## Architecture

```
┌──────────────────┐       ┌──────────────────┐       ┌──────────────────┐
│                  │       │                  │       │                  │
│   Next.js 16     │──────▸│   FastAPI         │──────▸│   InsForge       │
│   Frontend       │  API  │   Backend         │  REST │   (PostgreSQL,   │
│                  │       │                  │       │    Auth, Storage, │
│  Mapbox GL Map   │       │  Claude AI Chat   │       │    Realtime)     │
│  shadcn/ui       │       │  ReAct Agents     │       │                  │
│  InsForge SDK    │       │  Data Scrapers    │       │                  │
│                  │       │                  │       │                  │
└──────────────────┘       └──────────────────┘       └──────────────────┘
                                    │
                                    ▼
                           ┌──────────────────┐
                           │   TinyFish        │
                           │   (Police & News  │
                           │    Data Feeds)    │
                           └──────────────────┘
```

- **Frontend** calls the backend API at `localhost:8000` (configurable via `NEXT_PUBLIC_API_URL`)
- **Backend** proxies all database operations through InsForge's REST API via `InsForgeClient`
- **Auth** is handled by InsForge — JWT tokens issued on login/signup, validated server-side
- **Realtime** location sharing uses InsForge WebSockets — the backend receives webhook POSTs for DB persistence (zero WebSocket code in the backend)
- **AI Chat** uses a two-lane system: Lane 1 for simple location queries (fast, streaming), Lane 2 for complex people/group queries (ReAct loop with tool use) — both powered by Claude

### Backend API Routes

| Route             | Description                                     |
| ----------------- | ----------------------------------------------- |
| `/api/auth/`      | Signup, login, profile CRUD, onboarding         |
| `/api/incidents/` | Incident queries with geo/time/category filters |
| `/api/chat/`      | AI-powered safety chat                          |
| `/api/briefs/`    | Area safety briefs                              |
| `/api/reports/`   | Community incident reporting                    |
| `/api/groups/`    | Group CRUD, join via invite code                |
| `/api/places/`    | Saved places management                         |
| `/api/location/`  | Live location sharing, webhooks                 |
| `/api/geocode/`   | Forward/reverse geocoding                       |

### Database Schema

Key tables: `users`, `groups`, `group_members`, `saved_places`, `incidents`, `incident_sources`, `community_reports`, `locations_live`, `brief_cache`

Full schema defined in [`scripts/schema.sql`](scripts/schema.sql).

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm
- Python 3.11+

### Installation

```bash
# Clone the repo
git clone https://github.com/KshitijD21/CityWatch.git
cd CityWatch

# Install frontend dependencies
pnpm install

# Set up backend
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cd ..
```

### Environment Variables

Copy the example env files and fill in your keys:

```bash
cp frontend/.env.example frontend/.env.local
cp backend/.env.example backend/.env
```

**Frontend** (`frontend/.env.local`):

| Variable                        | Description                                        |
| ------------------------------- | -------------------------------------------------- |
| `NEXT_PUBLIC_INSFORGE_URL`      | InsForge project URL                               |
| `NEXT_PUBLIC_INSFORGE_ANON_KEY` | InsForge anonymous key                             |
| `NEXT_PUBLIC_API_URL`           | Backend API URL (default: `http://localhost:8000`) |

**Backend** (`backend/.env`):

| Variable            | Description                        |
| ------------------- | ---------------------------------- |
| `ANTHROPIC_API_KEY` | Anthropic API key for Claude       |
| `MAPBOX_TOKEN`      | Mapbox access token                |
| `TINYFISH_API_KEY`  | TinyFish API key for data sourcing |
| `INSFORGE_URL`      | InsForge project URL               |
| `INSFORGE_API_KEY`  | InsForge service role key          |

### Run

```bash
# Run both frontend and backend concurrently
pnpm dev

# Or run separately
pnpm dev:frontend   # Next.js at localhost:3000
pnpm dev:backend    # FastAPI at localhost:8000
```

### Seed & Scrape Data

```bash
pnpm seed              # Seed database with sample data
pnpm scrape:police     # Scrape police incident data
pnpm scrape:news       # Scrape news incident data
```

## Key Services

### InsForge

[InsForge](https://insforge.dev) is our Backend-as-a-Service providing:

- **PostgreSQL** database with REST API
- **Authentication** (email/password, OAuth) with JWT tokens
- **Realtime** WebSocket channels for live location sharing
- **Storage** for user-uploaded content

All database operations go through InsForge's REST API — no direct database connections.

### TinyFish

[TinyFish](https://tinyfish.app) powers our incident data pipeline:

- Aggregates police reports and news articles
- Provides structured incident data with geolocation
- Used by our scraper scripts to populate the incidents table

### Claude (Anthropic)

Claude powers the AI features:

- **Chat** — natural-language safety queries with streaming responses
- **Area Briefs** — automated safety summaries for any location
- **ReAct Agents** — multi-step reasoning for complex queries involving people, groups, and location cross-referencing
